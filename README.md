# 🌸 Flower — PHP Website

Сайт цветочного магазина на PHP, реализованный по прототипу.

## Структура файлов

```
flower/
├── index.php               # Главная страница (Hero)
├── css/
│   └── style.css           # Все стили
├── data/
│   └── products.php        # База данных товаров (массив)
├── includes/
│   ├── header.php          # Шапка + навигация (подключается везде)
│   └── footer.php          # Подвал + подписка
└── pages/
    ├── flowers.php         # Каталог букетов + поиск
    ├── contacts.php        # Контакты + форма обратной связи
    ├── delivery.php        # Доставка + форма заказа доставки
    ├── cart.php            # Корзина (изменение кол-ва, удаление, заказ)
    ├── wishlist.php        # Избранное
    └── subscribe.php       # Обработчик подписки
```

## Как запустить

### Вариант 1 — встроенный PHP-сервер
```bash
cd flower
php -S localhost:8000
```
Открыть: http://localhost:8000

### Вариант 2 — Apache/Nginx
Скопировать папку `flower/` в `htdocs/` (XAMPP) или `www/` (Denver/OpenServer).

## Реализованный функционал

- **Главная** — Hero с SVG-иллюстрацией, кнопки Shop / Find in Store
- **Каталог** — сетка из 6 букетов, поиск по названию/описанию, добавление в корзину и избранное
- **Корзина** — изменение количества, удаление, подсчёт итога, бесплатная доставка от $80, оформление заказа
- **Избранное** — список сохранённых букетов, перемещение всех в корзину
- **Контакты** — форма обратной связи (Full name / Email / Message)
- **Доставка** — форма заказа доставки, рабочие часы
- **Подписка** — форма в шапке и подвале (15% скидка)
- **Сессии PHP** — корзина и вишлист хранятся в $_SESSION

## Добавление новых товаров

В файле `data/products.php` добавьте элемент в массив `$products`:
```php
7 => [
    'id' => 7,
    'name' => 'Название букета',
    'desc' => 'Описание',
    'price' => 55,
    'emoji' => '🌼',
    'color' => '#fff9c4',
],
```

## В production нужно добавить

- Подключение к MySQL (PDO) вместо массива
- Отправку email через `mail()` или SMTP (PHPMailer)
- Сохранение подписчиков в БД
- Авторизацию пользователей
- Реальные фото букетов (`<img>` вместо emoji)
- Оплату (Stripe, ЮKassa и т.д.)
